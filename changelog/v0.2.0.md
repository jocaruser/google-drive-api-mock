# v0.2.0

Quality parity with illo3d and contract-fidelity hardening.

## Contract fidelity

- Drive resources and list envelopes carry `kind`
  (`drive#file`, `drive#fileList`) and `incompleteSearch`;
  Drive errors carry the legacy `errors[]` array
  (with `location` details on 401), Sheets errors stay
  `code`/`message`/`status` — matching the real per-API shapes.
- `values.update` responses include `updatedColumns`;
  spreadsheet envelopes include `spreadsheetUrl`.
- `values.clear` now clears exactly the requested range
  (cell, rectangle, column or row range, or the whole tab),
  as real Sheets does — previously any accepted range
  wiped the whole tab.
- `values.update` accepts bounded rectangles
  (payload must fit) alongside expandable cell anchors;
  reads of a bare cell return exactly that cell.
- Unencoded colons in ranges (`t!A1:B2`) are served;
  only Google's real RPC verbs (`:clear`, `:append`, …)
  dispatch as methods, and unmodelled ones 404 loudly.

## Serving external state

- A running server picks up files seeded (or a world reset)
  by another process on the same data directory.
  Freshness is judged by index content, not timestamps,
  so same-length rewrites cannot be missed.
- Every store mutator re-syncs first, so long-lived seeding
  stores never clobber server-authored state or reuse ids.
- A corrupt `_index.json` fails every request loudly
  (400 naming the file) and recovers once fixed;
  index read errors other than absence propagate.

## Standards

- ESLint, 100% coverage thresholds enforced in CI,
  dependency audit gate, Dependabot (npm, docker, actions),
  GHCR image publishing gated on the checks job,
  per-version changelog, one shared process bootstrap
  (`src/main.ts`) for bin, Docker and `pnpm start`.
