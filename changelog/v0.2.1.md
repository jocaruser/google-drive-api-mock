# v0.2.1

Fixes a real incident hit while wiring illo3d up to consume v0.2.0
as a git devDependency.

## Fixed: runaway installs when consumed as a git dependency

`package.json` wired its `build` script to the `prepare` lifecycle
hook, on the theory that consumers installing from git would always
get a fresh `dist/`. In practice, a `prepare` script on a package
with its own devDependencies forces pnpm to build that package in
isolation for *every* consumer pulling it as a git dependency — which
means fetching and installing this repo's entire devDependency set
from scratch, every install, for every consumer. Under real-world
network flakiness that isolated install can itself stall and retry,
and in testing it spiralled into dozens of nested `pnpm install`
processes before being killed by hand.

`dist/` is already committed and CI verifies it matches sources on
every push (`git diff --exit-code dist`), so no consumer ever needed
a build step at all. `prepare` is removed; `pnpm install` on this
repo is now a plain devDependency install, and `pnpm run build` is
an explicit step contributors run (and CI enforces) after touching
`src/`. Installing this package as a git dependency is now a pure
file copy, as originally intended.

See ADR-0003 (updated) for the full account.
